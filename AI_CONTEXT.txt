=== ROUTEMATE PROJECT CONTEXT ===
App: Flutter motorcycle group safety app
Backend: Firebase (Firestore + Realtime DB + Cloud Functions + FCM)
Maps: Mapbox Flutter SDK
Weather: OpenWeatherMap API (via Cloud Function proxy)
State management: Riverpod (hoặc Provider - tùy nhóm chốt)
Target: Android 8+ / iOS 13+
Key constraints: No hardcoded API keys, Distance Filter 50m for GPS, Haversine for distance calc
Firebase schema: Firestore(rooms/{roomId}: {members, fcmTokens}, alerts/{alertId}: {lat, lng, type, createdAt})
Realtime DB schema: gps/{roomId}/{memberId}: {lat, lng, updatedAt}



Mô tả dự án và cấu trúc thư mục hoàn chỉnh: 
Ý tưởng: ứng dụng sẽ có 3 tab tính năng như đã nói: tab đầu tiên sẽ là tính năng 1: định tuyến: ở tính năng này sẽ giống như ứng dụng gg map,thể hiện bản đồ bằng APi MAPBOX thấy được vị trí người dùng hiện tại. có thể chọn được điểm đến. từ đó có thể tính được khoảng cách, thời gian di chuyển bằng xe. Nếu bấm vào nút bắt đầu di chuyển thì sẽ vẽ cụ thể lộ trình polyline để hiển thị đường đi. Và trên đường đó sẽ có marker thời tiết được lấy từ API weather map, Cùng với đó là tích hợp AI mượn từ api gemini flash để người dùng có thể dùng giọng nói để gán cảnh báo trên đường đi cho các thành viên trong đoàn. Tab thứ 2 sẽ là tính năng Group Radar. khi mở tab này thì sẽ hiện lên các thông tin nhập vào: tên/id; Nhập ID muốn vào; và 2 mode: vào phòng hoặc tạo phòng; chọn role (leader, member, sweeper). Tôi muốn tính năng 2 này phải liên quan đến tính năng 1, khi vào phòng hoặc tạo phòng phải đều mang đầy đủ các tính năng của tính năng 1. và có thêm các chức năng mới là hiển thị các thành viên trên bản đồ chứ không chỉ mình ta như tính năng 1. Cũng phải hiển thị được tất cả marker mà leader báo cáo lên trên tuyến đường. Tính năng số 3 thì sẽ có 1 nút nhấn SOS như tôi đã nói.
